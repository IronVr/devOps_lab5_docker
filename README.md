# devOps_lab5_docker

## Подготовка среды и сетевая настройка
* Работа выполнялась на виртуальной машине с Ubuntu (VirtualBox).
* Для доступа к веб-серверу с хост-машины настроен **проброс портов** в VirtualBox:
  - Тип сети: NAT
  - Проброшен порт хоста `54321` на порт гостя `54321`
* На гостевой машине установлен Docker `sudo apt install docker.io`
---
## Создание структуры проекта

В домашней директории создана следующая структура файлов и каталогов:

     ```
     .
     ├── deploy.sh
     ├── nginx/
     │   ├── dockerfile
     │   ├── index.html
     │   └── nginx-conf/
     │     └── nginx.conf
     ```

2. **Разработка:**
   * **nginx.conf** — конфигурация сервера Nginx, прослушивающего порт 80 и отдающего статические файлы.
   * **dockerfile** — инструкция для сборки образа на базе Nginx, копирования HTML-страницы и конфигурации, проверки конфига и запуска.
   * **deploy.sh** — скрипт для сборки образа, запуска контейнера с пробросом порта и проверки.

3. **Запуск:**
   * Необходимо выдать права на исполнение 'chmod +x deploy.sh' и запустить скрипт './deploy.sh'

4. **Проверка результатов:**

   * Проверена доступность страницы с помощью `curl`:
     ```bash
     curl 127.0.0.1:54321
     ```
   * Или с хост-машины после настройки проброса портов страница доступна по адресу 'http://localhost:54321'
